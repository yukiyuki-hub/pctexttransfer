# pctexttransfer

## プロジェクト概要

PC間のテキスト転送ツール。

## Git 運用ルール

### コード変更時は必ずGitHubにプッシュする

コードを変更・追加・削除したら、毎回以下の手順でGitHubにプッシュすること。

```bash
git add <変更ファイル>
git commit -m "変更内容を簡潔に説明するメッセージ"
git push origin main
```

- `git add .` や `git add -A` は使わない。変更したファイルを名指しで追加する。
- コミットメッセージは日本語でも英語でもよい。何を変えたかが伝わる内容にする。

### リモートリポジトリ

- URL: https://github.com/yukiyuki-hub/pctexttransfer.git
- ブランチ: `main`

## 絶対にコミットしてはいけないファイル

以下のファイル・フォルダは `.gitignore` で除外済み。絶対にコミット・プッシュしないこと。

| ファイル/フォルダ | 理由 |
|---|---|
| `.env` | APIキー・パスワードなどの機密情報が含まれる。GitHubに上げると世界中から見られてしまう |
| `node_modules/` | 外部ライブラリの大量ファイル（数十〜数百MB）。毎回プッシュするとリポジトリが重くなる |

機密情報は `.env` に書き、`.env.example` にキー名だけ書いてコミットする。
